Edit the `setup` file to add two new installation blocks BEFORE the `exit 0` line (line 32). Keep all existing content unchanged.

**Add jq installation block** (insert after the alacritty block, before exit 0):

```bash
# jq — required by ralph agent loop for parsing prd.json
if ! command -v jq >/dev/null; then
  apt-get install -y jq 2>/dev/null || brew install jq 2>/dev/null || echo "WARNING: jq not installed — ralph requires jq"
fi
```

This tries apt-get first (DevPod containers are typically Ubuntu/Debian), falls back to brew (macOS), and warns if neither works.

**Add Claude Code CLI installation block** (insert after the jq block, before exit 0):

```bash
# Claude Code CLI — native installer places binary at ~/.local/bin/claude
# Must cd to /tmp first to prevent filesystem scan hang in containers
if ! command -v claude >/dev/null; then
  cd /tmp && curl -fsSL https://claude.ai/install.sh | bash
fi
```

Key details:
- The `cd /tmp` is CRITICAL — without it, the installer scans from the current directory and hangs in containers (see Research Pitfall 2)
- The `command -v claude` guard makes it idempotent — skip if already installed
- The native installer is self-contained, does not require Node.js at install time, and auto-updates
- The binary lands at `~/.local/bin/claude` which is already in PATH via dot_zshrc

Do NOT:
- Add `sudo` to the curl command (causes permission issues)
- Add claude to mise/config.toml (mise is per-project; setup.sh handles global tools)
- Modify any existing blocks in the setup script
- Remove `exit 0` at the end
