    **File structure (4 major sections):**

    1. **Overview: Ralph as a Context Engineering System (150 words)**
       - Context engineering: The discipline of managing information flow, context boundaries, and knowledge persistence in AI systems (2025-2026 industry standard)
       - Ralph implements context engineering through:
         - **Progressive Disclosure**: Each iteration receives only essential context (no session history)
         - **Identity Management**: Agent assumes specific role per iteration (via CLAUDE.md instructions)
         - **Persistent State**: Knowledge stored externally (progress.txt, git, CLAUDE.md files) not in prompts

    2. **Sub-Agent Spawning Pattern (300-400 words)**
       - **Pattern Definition:**
         - Decompose large task into subtasks
         - Each subtask handled by fresh agent (clean context)
         - Agents coordinate via shared state (files, messages)
         - Prevents "context drift" from earlier subtasks biasing later ones

       - **Why It Matters:**
         - Token efficiency: Each agent starts lean, doesn't carry bloat from previous work
         - Decision quality: Later subtasks not biased by earlier context
         - Scalability: Can chain many subtasks without token explosion
         - Auditability: Each agent's work is isolated and traceable

       - **How Ralph Implements It:**
         - Subtasks: Individual user stories from prd.json
         - Fresh spawning: `claude --no-session-persistence` (scripts/ralph line 104)
         - Shared state: progress.txt (learnings), prd.json (task list), git (code)
         - Coordination: CLAUDE.md reads prd.json, progress.txt in sequence

       - **Comparison to Other Approaches:**
         - ❌ Single long context: Context balloons, decisions degrade
         - ❌ Simple session reuse: Agent carries assumptions from earlier work
         - ✅ Sub-agent spawning: Fresh per subtask, coordination via shared state

    3. **Knowledge Persistence: Lab Notebook Pattern (300-350 words)**
       - **Pattern Definition (from Research):**
         - Multi-agent systems maintain shared "lab notebook" (state file)
         - Tracks: what was tried, what worked (patterns), what failed (gotchas), dependencies
         - Each agent reads lab notebook before working
         - Each agent appends findings after working
         - Creates unified team knowledge without explicit communication

       - **How Ralph Implements It:**
         - Lab notebook: progress.txt
         - Structure: Consolidated Codebase Patterns section + per-iteration sections
         - Read step: CLAUDE.md Step 4 (critical)
         - Write step: Claude appends learnings to progress.txt

       - **Why This Works:**
         - No API calls needed (file-based, works offline)
         - Clear format (markdown) readable by humans and AI
         - Non-destructive (append-only, preserves history)
         - Self-documenting (patterns accumulate, forming knowledge base)

       - **Fragility & Risk:**
         - Single point of failure: If Step 4 is removed, persistence breaks
         - Format sensitivity: If format deviates, future Claude can't parse it
         - Manual maintenance: Patterns need periodic review and consolidation

    4. **Progressive Disclosure & Selective Context Loading (250-300 words)**
       - **Principle:**
         - Load only necessary context for current task
         - Keep baseline instructions light
         - Reveal details only when needed

       - **How Ralph Implements It:**
         - Baseline: CLAUDE.md provides agent instructions (task, role, format)
         - Per-story context: Claude reads prd.json to find current story
         - Learnings context: Claude reads progress.txt Codebase Patterns (only relevant patterns, not full history)
         - Module context: Claude reads CLAUDE.md files in directories being modified (discoverability)

       - **Benefit:**
         - Keeps context lean (token efficient)
         - Reduces noise (Claude focuses on current task, not all prior work)
         - Allows scaling (can accumulate many patterns without bloating each iteration)

       - **Comparison to Alternative:**
         - ❌ Dump all context: "Here's progress.txt, here's all git history, here's all CLAUDE.md files"
           - Token bloat, context confusion, harder for Claude to prioritize
         - ✅ Progressive disclosure: "Read prd.json to find your task, then read Codebase Patterns from progress.txt"
           - Lean context, clear priority, scalable

    5. **State Transitions & Completion Tracking (200-250 words)**
       - **Pattern:**
         - State machines in multi-agent systems use explicit markers
         - Ralph uses: prd.json `passes: true/false`, `<promise>COMPLETE</promise>` sentinel

       - **How Ralph Uses It:**
         - Story state: `passes: false` = incomplete, `passes: true` = done
         - Overall completion: `<promise>COMPLETE</promise>` in Claude output signals "all done"
         - Why sentinels work: Unambiguous, parseable by scripts, reliable

       - **Why Not Use Alternative Approaches:**
         - ❌ Exit codes: Unreliable (script exits for many reasons)
         - ❌ File timestamps: Ambiguous (many changes, hard to detect "completion")
         - ❌ Predefined completion markers: Easy to forget to output
         - ✅ Explicit XML sentinel: Unambiguous, easy to grep, works across tools

       - **Claude's Role:**
         - Must output `<promise>COMPLETE</promise>` when ALL prd.json stories have `passes: true`
         - Scripts/ralph greps for this sentinel to detect completion
         - If Claude doesn't output it, loop continues (max_iterations acts as safety brake)

    **Tone & Style:**
    - Academic-adjacent but still practical
    - Use industry terminology (progressive disclosure, lab notebook, sub-agent spawning)
    - Explain tradeoffs (why this approach vs. alternatives)
    - Reference that these are established patterns from 2025-2026 AI engineering research
    - Assume reader understands distributed systems or multi-agent concepts
    - But don't assume they know Ralph yet
